# BFS (Breadth-First Search)

## 정의
**그래프/트리에서 가장 가까운 노드부터 탐색하는 알고리즘**
- 너비 우선 탐색
- 최단 거리 보장 (가중치 동일 경우)

---
## 특징
- Queue 사용
- 시작점에서 "거리" 개념 존재
- 최단 거리(간선 가중치 동일) 보장

---
## 알고리즘 구조 (3단계)
```
1. 초기화 (Initialize)
   └─ visited[] 초기화, 시작 노드를 큐에 삽입

2. 반복 탐색 (Expansion)
   └─ 큐에서 꺼냄 → 인접 노드 확인 → 조건 검사 → 큐에 추가

3. 종료 조건 (Goal Check)
   └─ 원하는 지점 도달 or 큐 비면 종료
```

---
## 최단거리 보장 조건 (필수 2가지)

### 1. 가중치 동일성 (Uniform Weight)
**모든 간선의 가중치가 동일해야 함**
- 가중치 없음 (모두 1)
- 모든 가중치가 같음
- ⚠️ 다른 경우 → 다익스트라 사용

### 2. 명확한 탐색 목표 또는 범위
**탐색할 영역이 명확해야 함**
- 목표 노드가 정해짐
- 또는 모든 노드 탐색이 목표
- 범위 제한 있음 (2D grid, n×m)

⚠️ **1번을 만족하지 않으면 최단거리 보장 불가!**

---
## 언제 사용하는가?

### ✅ 사용 가능 신호
- 최단 거리 (가중치 동일)
- 레벨 단위 탐색
- 최단 전파/전염/확산
- 2D grid 최단 경로

### ❌ 사용 불가 신호
- 간선 가중치가 존재 (→ 다익스트라)
- 깊이 우선 탐색이 유리한 문제
- 모든 경우를 순회해야 할 때 (→ DFS)

---
## 대표 예제
- 미로 최단 거리
- 토마토 / 불 번짐 / 전염 (퍼지기 문제)
- 최단 거리 그래프 탐색
- 섬 개수/영역 탐색

---
## 시간복잡도
- **그래프**: O(V + E)
- **2D grid**: O(N × M)

---
## vs 다른 알고리즘

| | BFS | DFS | 다익스트라 |
| ------ | ---------- | -------------- | -------------- |
| 탐색 방식 | 너비 우선 | 깊이 우선 | 가중치 고려 |
| 자료구조 | Queue | Stack/재귀 | 우선순위 큐 |
| 가중치 | 동일만 | 무관 | 가능 |
| 최단거리 | ✅ | ❌ | ✅ |
| 시간복잡도 | O(V+E) | O(V+E) | O(E log V) |

---
## 핵심 정리
1. **Queue 기반**: 순서대로 처리
2. **최단거리 보장**: 가중치가 동일할 때만
3. **레벨 탐색**: 거리별로 체계적 탐색
4. **주의**: 가중치 있으면 다익스트라 사용
