# 그리디 알고리즘

## 정의

**매 단계에서 지역적 최적 선택을 통해 전역 최적해를 구하는 알고리즘**

- 현재 가장 좋은 것만 선택
- 선택 후 번복 안 함
- 미래 고려 안 함

---

## 알고리즘 구조 (3단계)

```
1. 선택 절차 (Selection)
   └─ 현재 상태에서 최적이라고 생각되는 것 선택

2. 적절성 검사 (Feasibility)
   └─ 선택이 문제 조건을 만족하는지 검사

3. 해답 검사 (Solution Check)
   └─ 문제가 해결되었는지 확인
```

---

## 최적해 보장 조건 (필수 2가지)

### 1. 탐욕적 선택 속성 (Greedy Choice Property)

**지역 최적 선택이 전역 최적해로 이어짐**

- 각 단계의 선택이 이후 선택에 영향 없음
- 한 번 선택하면 번복 불필요

### 2. 최적 부분 구조 (Optimal Substructure)

**전체 최적해 = 부분 문제들의 최적해**

- 부분 문제를 최적으로 풀면 전체도 최적
- DP와 공통 속성

**⚠️ 둘 다 만족해야 그리디로 최적해 보장!**

---

## 언제 사용하는가?

### ✅ 사용 가능 신호

- 최대/최소를 구하는 문제
- 정렬 후 순차 선택으로 해결 가능
- 직관적으로 "당연히 이게 최선"
- 각 선택이 독립적

### ❌ 사용 불가 신호

- 반례가 쉽게 떠오름
- 이전 선택이 이후 선택에 영향
- 모든 경우를 따져봐야 할 것 같음

---

## 대표 예제

### Kruskal (MST)

| 단계   | 내용                            |
| ------ | ------------------------------- |
| 선택   | 가장 작은 가중치 간선           |
| 검사   | 사이클 생성하는가? (Union-Find) |
| 종료   | 간선 V-1개                      |
| 복잡도 | O(E log E)                      |

### 회의실 배정

| 단계   | 내용                     |
| ------ | ------------------------ |
| 선택   | 종료 시간 가장 빠른 회의 |
| 검사   | 이전 회의와 겹치는가?    |
| 종료   | 더 이상 선택 불가        |
| 복잡도 | O(n log n)               |

---

## vs 다른 알고리즘

|        | 그리디     | DP             | 완전탐색       |
| ------ | ---------- | -------------- | -------------- |
| 선택   | 지금 최선  | 모든 경우 비교 | 모든 경우 시도 |
| 번복   | 불가       | 가능           | -              |
| 복잡도 | O(n log n) | O(n²~)         | O(2ⁿ~)         |
| 최적해 | **조건부** | 항상           | 항상           |

---

## 실패 사례: 0-1 배낭

```
용량 50kg
물건: (가치, 무게) = [(60,10), (100,20), (120,30)]

그리디(가치/무게): 60+100 = 160
실제 최적: 100+120 = 220 ❌

→ 탐욕적 선택 속성 불만족
```

---

## 시간복잡도

- **정렬 필요**: O(n log n)
- **순회만**: O(n)
- **우선순위 큐**: O(n log n)

---

## 핵심 정리

1. **3단계**: 선택 → 검사 → 종료
2. **2조건**: 탐욕적 선택 속성 + 최적 부분 구조
3. **특징**: 빠르지만 조건 만족해야 최적해
4. **주의**: 반드시 증명 필요 (교환 논증 등)
