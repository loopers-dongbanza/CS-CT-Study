# BackTracking (백트래킹)

## 정의
**DFS 기반으로 가지치기를 통해 불가능한 경로를 조기 차단하는 알고리즘**
- 모든 가능성을 탐색하되 조건 위반 시 즉시 중단
- "선택 → 검사 → 복원" 구조

---
## 특징
- 가지치기 (Pruning) - 불필요한 경로 조기 중단
- 선택 복원 가능 (상태 복구)
- NP 문제 해결에 유용

---
## 알고리즘 구조 (4단계)
```
1. 선택 (Choose): 가능한 선택지 선택
2. 탐색 (Explore): 재귀적으로 다음 단계 탐색
3. 검사 (Check): 조건 만족 여부 판단
4. 복원 (Unchoose): 선택 복원 후 다른 경로 탐색
```

---
## 효율적 동작 조건 (필수 2가지)

### 1. 선택 복원 (Unchoose)
**선택 후 복원 가능한 구조**
- 상태 변경이 가역적

### 2. 가지치기 조건 (Pruning)
**불가능한 경로를 조기 차단**
- 없으면 부분집합과 동일 (O(2^N))
- ⚠️ 가지치기가 약하면 시간초과!

---
## 언제 사용하는가?

### ✅ 사용 가능 신호
- 조합 문제 (조건부 경로)
- N과M (순열, 조합)
- 제약조건 문제 (N-Queen, 스도쿠)

### ❌ 사용 불가 신호
- 가지치기 조건이 없음 (부분집합 사용)
- 최단경로 문제 (BFS 사용)

---
## 대표 문제
- 조합 (로또 문제)
- N-Queen 문제
- 스도쿠
- 순열 (N과M)

---
## 시간복잡도
**정확한 계산 불가**
- 가지치기 조건에 따라 크게 달라짐
- 최악의 경우: O(2^N) (가지치기 없을 때)


---
## 핵심 정리
1. **가지치기가 핵심**: 불가능한 경로는 조기 차단
2. **선택 복원 필수**: Unchoose 단계 중요
3. **효율성**: 부분집합보다 훨씬 효율적
4. **주의**: 가지치기 조건이 약하면 TLE!
